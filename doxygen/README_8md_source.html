8 C-library for calculating Solvent Accessible Surface Areas.
9 
10 License: MIT (see file LICENSE). Copyright: Simon Mitternacht 2013-2016.
11 
12 FreeSASA is a C library and command line tool for calculating Solvent
13 Accessible Surface Area (SASA) of biomolecules. It is designed to be
14 simple to use with defaults, but allows customization of all
15 parameters of the calculation and provides a few different tools to
16 analyze the results. Python bindings are also included in the
17 repository.
18 
19 By default Lee & Richards' algorithm is used, but Shrake & Rupley's is
20 also available. Both can be parameterized to arbitrary precision, and
21 for high resolution versions of the algorithms, the calculations give
22 identical results.
23 
24 FreeSASA assigns a radius and a class to each atom. The atomic radii
25 are by default the _ProtOr_ radii defined by Tsai et
26 al. ([JMB 1999, 290: 253](http://www.ncbi.nlm.nih.gov/pubmed/10388571))
27 for standard amino acids and nucleic acids, and the van der Waals
28 radius of the element for other atoms. Each atom is also classified as
29 either polar or apolar.
30 
31 Users can provide their own atomic radii and classifications via
32 configuration files. The input format for configuration files is
33 described in the
34 [online documentation](http://freesasa.github.io/doxygen/Config-file.html),
35 and the `share/` directory contains some sample configurations,
36 including one for the NACCESS parameters
37 ([Hubbard & Thornton 1993](http://www.bioinf.manchester.ac.uk/naccess/)).
38 
39 Version 2.0 adds some new features and breaks a few parts of the
40 interface from 1.x (mainly the API), see CHANGELOG.md for detailed
41 information.

43 Building and installing
44 ------------------------
45 
46 FreeSASA can be compiled and installed using the following
47 
48  ./configure
49  make && make install
50 
51 NB: If the source is downloaded from the git repository the
52 configure-script needs to be set up first using `autoreconf -i`. Users
53 who don't have autotools installed, can download a tarball that
54 includes the autogenerated scripts from http://freesasa.github.io/ or
55 from the latest
56 [GitHub-release](https://github.com/mittinatten/freesasa/releases).
57 
58 The above commands build and install the command line tool `freesasa`
59 (built in `src/`), the commands
60 
61  freesasa -h
62 
63 and, if installed,
64 
65  man freesasa
66 
67 give an overview of options. To run a calculation from PDB-file input
68 using the defaults, simply type
69 
70  freesasa <pdb-file>
71 
72 In addition, `make install` installs the header `freesasa.h` and the
73 library `libfreesasa`.
74 
75 The configuration can be changed with these options:
76 * `--disable-json` build without support for JSON output.
77 * `--disable-xml` build without support for XML output.
78 * `--disable-threads` build without multithreaded calculations
79 * `--enable-doxygen` activates building of Doxygen documentation
80 
81 For developers:
82 * `--enable-check` enables unit-testing using the Check framework
83 * `--enable-gcov` adds compiler flags for measuring coverage of tests
84  using gcov
85 * `--enable-parser-generator` rebuild parser/lexer source from
86  Flex/Bison sources (the autogenerated code is included in the
87  repository, so no need to do this if you are not going to change
88  the parser).

90 Python module
91 -------------
92 
93 The Python bindings are available from PyPi and can be installed using
94 
95  pip install freesasa
96 
97 This module is found in a separate repository
98 https://github.com/freesasa/freesasa-python
99 (the bindings included here will eventually be removed).
100 The PyPi module has binaries for Mac OS X and Windows, for a number
101 of Python versions.

103 Documentation

117 Compatibility and dependencies
118 ------------------------------
119 
120 The program has been tested successfully with several versions of GNU
121 C Compiler and Clang/LLVM. The library can be built using only
122 standard C and GNU libraries. The standard build depends on
123 [json-c](https://github.com/json-c/json-c) and
124 [libxml2](http://xmlsoft.org/). These can be disabled by configuring
125 with `--disable-json` and `--disable-xml` respectively.
126 
127 Developers who want to do testing need to install the Check unit
128 testing framework. Building the full reference manual requires Doxygen
129 (version > 1.8.8). Building the Python bindings requires
130 Cython. Changing the selection parser and lexer requires Flex and
131 Bison. These build options, which add extra dependencies, are disabled
132 by default to simplify installation for users only interested in the
133 command line tool and and/or C Library.
