11 C-library for calculating Solvent Accessible Surface Areas.
12 
13 License: MIT (see file LICENSE). Copyright: Simon Mitternacht 2013-2016.
14 
15 FreeSASA is a C library and command line tool for calculating Solvent
16 Accessible Surface Area (SASA) of biomolecules. It is designed to be
17 very simple to use with defaults, but allows customization of all
18 parameters of the calculation and provides a few different tools to
19 analyze the results as well. Python bindings are also included in the
20 repository.
21 
22 The library includes both the algorithm by Lee & Richards and that by
23 Shrake & Rupley. Verification has been done by comparing the results
24 of the two calculations and by visual inspection of the surfaces found
25 by them (and comparing with analytic results in the two-atom
26 case). Both can be parameterized to arbitrary precision, and for high
27 resolution versions of the algorithms, the calculations give identical
28 results.
29 
30 FreeSASA assigns a radius and a class to each atom. The atomic radii
31 are by default those defined by Tsai et al. ([JMB 1999, 290:
32 253](http://www.ncbi.nlm.nih.gov/pubmed/10388571)) for standard amino
33 acids and nucleic acids, and the van der Waals radius of the element
34 for other atoms. Each atom is also assigned to a class. The default
35 classes are `Apolar` (carbon) and `Polar` (all other elements). The
36 program outputs the total SASA and the area of these two classes, if a
37 protein has more than one chain the contribution of each chain is also
38 included in the output.
39 
40 Users can provide their own atomic radii and classes, either via
41 configuration files or via the API. The input format for configuration
42 files is described in the [online
43 documentation](http://freesasa.github.io/doxygen/Config-file.html),
44 and the `share/` directory contains some sample configurations,
45 including one for the NACCESS parameters ([Hubbard & Thornton
46 1993](http://www.bioinf.manchester.ac.uk/naccess/)).

48 Compiling and installing
49 ------------------------
50 
51 FreeSASA can be compiled and installed using the following
52 
53  ./configure
54  make && make install
55 
56 NB: If the source is downloaded from the git repository the
57 configure-script needs to be set up first using `autoreconf -i`. Users
58 who don't have autotools installed, can download a tarball that
59 includes the autogenerated scripts from http://freesasa.github.io/ or
60 from the latest
61 [GitHub-release](https://github.com/mittinatten/freesasa/releases).
62 
63 The above commands build and install the command line tool `freesasa`
64 (built in `src/`), the command
65 
66  freesasa -h
67 
68 gives an overview of options. To run a calculation from PDB-file input
69 using the defaults, simply type
70 
71  freesasa <pdb-file>
72 
73 `make install` also installs the header `freesasa.h` and the library
74 `libfreesasa`. If the configure script is called with the option
75 `--enable-python-bindings`, the Python module is also built and
76 installed.
77 
78 Configuring with
79 
80  ./configure CFLAGS="-Ofast"
81 
82 increases the speed of the Shrake & Rupley algorithm significantly
83 (10-20%), as compared to the standard "-O2". There seems to be no
84 measurable effect on Lee & Richards.
85 
86 The configuration script can be customized with general options:
87 * `--enable-python-bindings` builds Python bindings, requires Cython
88 * `--with-python=<python>` specifies which python binary to use
89 * `--disable-threads` Build without multithreaded calculations
90 * `--enable-doxygen` activates building of Doxygen documentation
91 
92 And some options for developers:
93 * `--enable-check` enables unit-testing using the Check framework
94 * `--enable-gcov` adds compiler flags for measuring coverage of tests
95  using gcov
96 * `--enable-parser-generator` rebuild parser/lexer source from
97  Flex/Bison sources (the autogenerated code is included in the
98  repository, so no need to do this if you are not going to change
99  the parser).

101 Documentation
102 -------------
103 
104 Enabling Doxygen builds a [full reference
105 manual](http://freesasa.github.io/doxygen/), documenting both CLI and
106 API, also available on the web at http://freesasa.github.io/.
107 
108 After building the package, calling
109 
110  freesasa -h
111 
112 explains how the commandline tool can be used.

114 Compatibility and dependencies
115 ------------------------------
116 
117 The program has been tested successfully with several versions of GNU
118 C Compiler and Clang/LLVM. Building the library only requires standard
119 C and GNU libraries. Developers who want to do testing need to install
120 the Check unit testing framework. Building the full reference manual
121 requires Doxygen (version > 1.8.8). Building the Python bindings
122 requires Cython. Changing the selection parser and lexer requires Flex
123 and Bison. These build options, which add extra dependencies,
124 are disabled by default to simplify installation for users only
125 interested in the command line tool and and/or C Library.
