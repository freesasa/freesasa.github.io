23 @page CLI Command-line Interface
24 
25 Building FreeSASA creates the binary `freesasa`, which is installed by
26 `make install`. Calling
27 
28  $ freesasa -h
29 
30 displays a help message listing all options. The following text

33 @section CLI-Default Run using defaults
34 
35 In the following we will use the RNA/protein complex PDB structure
36 3WBM as an example. It has four protein chains A, B, C and D, and two
37 RNA strands X and Y. To run a simple SASA calculation using default
38 parameters, simply type:
39 
40  $ freesasa 3wbm.pdb
41 
42 This generates the following output
43 
44  ## FreeSASA 2.0 ##
45 
46  PARAMETERS
47  algorithm : Lee & Richards
48  probe-radius : 1.400
49  threads : 2
50  slices : 20
51 
52  INPUT
53  source : 3wbm.pdb
54  chains : ABCDXY
55  atoms : 3714
56 
57  RESULTS (A^2)
58  Total : 25190.77
59  Apolar : 11552.38
60  Polar : 13638.39
61  CHAIN A : 3785.49
62  CHAIN B : 4342.33
63  CHAIN C : 3961.12
64  CHAIN D : 4904.30
65  CHAIN X : 4156.46
66  CHAIN Y : 4041.08
67 
68 The results are all in the unit Ångström-squared.

70 @section parameters Changing parameters
71 
72 If higher precision is needed, the command
73 
74  $ freesasa -n 100 3wbm.pdb
75 
76 specifies that the calculation should use 100 slices per atom instead of
77 the default 20. The command
78 
79  $ freesasa --shrake-rupley -n 200 --probe-radius 1.2 --n-threads 4 3wbm.pdb
80 
81 instead calculates the SASA using Shrake & Rupley's algorithm with 200
82 test points, a probe radius of 1.2 Å, using 4 parallel threads to
83 speed things up.
84 
85 If the user wants to use their own atomic radii the command
86 
87  $ freesasa --config-file <file> 3wbm.pdb
88 
89 Reads a configuration from a file and uses it to assign atomic
90 radii. The program will halt if it encounters atoms in the PDB input
91 that are not present in the configuration. See @ref Config-file for
92 instructions how to write a configuration.
93 
94 To use the atomic radii from NACCESS call
95 
96  $ freesasa --radii=naccess 3wbm.pdb
97 
98 Another way to specify a custom set of atomic radii is to store them as
99 occupancies in the input PDB file
100 
101  $ freesasa --radius-from-occupancy 3wbm.pdb
102 
103 This option allows the user to first use the option `--format=pdb` (see @ref CLI-PDB) to
104 write generate a PDB file with the radii used in the calculation,
105 modify the radii of individual atoms in that file, and then recalculate
106 the SASA with these modified radii.

109 @section Output Output formats
110 
111 In addition to the standard output format above FreeSASA can export
112 the results as @ref CLI-JSON, @ref CLI-XML, @ref CLI-PDB, @ref
113 CLI-RSA, @ref CLI-RES and @ref CLI-SEQ using the option
114 `--format`. The level of detail of JSON and XML output can be
115 controlled with the option `--output-depth=<depth>` which takes the
116 values `atom`, `residue`, `chain` and `structure`. If `atom` is
117 chosen, SASA values are shown for all levels of the structure,
118 including individual atoms. With `chain`, only structure and chain
119 SASA values are printed (this is the default).
120 
121 The output can include relative SASA values for each residues. To
122 calculate these a reference SASA value is needed, calculated using the
123 same atomic radii. At the moment such values are only available for
124 the ProtOr and NACCESS radii (selected using the option `--radii`), if
125 other radii are used relative SASA will be excluded (in RSA output all
126 REL columns will have the value 'N/A').
127 
128 The reference SASA values for residue X are calculated from Ala-X-Ala
129 peptides in a stretched out configuration. The reference
130 configurations are supplied for reference in the directory
131 `rsa`. Since these are not always the most exposed possible
132 configuration, and because bond lengths and bond angles vary, the
133 relative SASA values will sometimes be larger than 100 %. At the
134 moment there is no interface to supply user-defined reference values.

346 @subsubsection RSA-naccess Using the NACCESS configuration
347 
348 The reference values for the NACCESS configuration in FreeSASA are not
349 exactly the same as those that ship with NACCESS, but have been
350 calculated from using the same Ala-X-Ala tripeptides (kindly donated by
351 the creators of NACCESS). By default NACCESS defines amino acid CA
352 atoms as side-chain, whereas FreeSASA defines them as main-chain. The
353 flag `-b` defines CA as main-chain in NACCESS.
354 
355 Calling
356 
357  $ freesasa 3wbm.pdb -n 20--format=rsa --radii=naccess
358 
359 will give an RSA file where the ABS columns should be identical to
360 NACCESS with flag `-b`. All REL values will differ slightly because of
361 round-off errors in the references. The difference will be large
362 for side-/main-chain REL values, since the FreeSASA references were
363 calculated with CA as main-chain. NACCESS also gives different results
364 for the nucleic acid main-chain and side-chain. FreeSASA defines the
365 (deoxy)ribose and phosphate groups as main-chain and the base as
366 side-chain.
367 
368 Future versions might allow specifying what atoms are main-chain
369 through the configuration file, and letting the user set their own
370 reference values.

372 @section CLI-select Selecting groups of atoms
373 
374 The option `--select` can be used to define groups of atoms whose
375 integrated SASA we are interested in. It uses a subset of the Pymol
376 `select` command syntax, see @ref Selection for full
377 documentation. The following example shows how to calculate the sum of
378 exposed surface areas of all aromatic residues and of the four chains
379 A, B, C and D (just the sum of the areas above).
380 
381  $ freesasa --select "aromatic, resn phe+tyr+trp+his+pro" --select "abcd, chain A+B+C+D" 3wbm.pdb
382  ...
383  SELECTIONS
384  freesasa: warning: Found no matches to resn 'TRP', typo?
385  freesasa: warning: Found no matches to resn 'HIS', typo?
386  aromatic : 1196.45
387  abcd : 16993.24
388 
389 The lines shown above are appended to the regular output. This
390 particular protein did not have any TRP or HIS residues, hence the
391 warnings (written to stderr). The warnings can be supressed with the
392 flag `-w`.

464 @section Input PDB input
465 
466 @subsection Hetatom-hydrogen Including extra atoms
467 
468 The user can ask to include hydrogen atoms and HETATM entries in the
469 calculation using the options `--hydrogen` and `--hetatm`. In both
470 cases adding unknown atoms will emit a warning for each atom. This can
471 either be amended by using the flag `-w` to suppress warnings, or by
472 using a custom classifier so that they are recognized (see @ref
473 Config-file).
474 
475 @subsection Halt-skip Skipping unknown atoms
476 
477 By default FreeSASA guesses the element of an unknown atom and uses
478 that elements VdW radius. If this fails the radius is set to 0 (and
479 hence the atom will not contribute to the calculated area). Users can
480 request to either skip unknown atoms completely (i.e. no guessing) or
481 to halt when unknown atoms are found and exit with an error. This is
482 done with the option `--unknown` which takes one of the three
483 arguments `skip`, `halt` or `guess` (default). Whenever an unknown
484 atom is skipped or its radius is guessed a warning is printed to
485 stderr.
486 
487 @subsection Chains-models Separating and joining chains and models
488 
489 If a PDB file has several chains and/or models, by default all chains
490 of the first model are used, and the rest of the file is ignored. This
491 behavior can be modified using the following options
492 
493  - `--join-models`: Joins all models in the input into one large
494  structure. Useful for biological assembly files were different
495  locations of the same chain in the oligomer are represented by
496  different MODEL entries.
497 
498  - `--separate-models`: Calculate SASA separately for each model in
499  the input. Useful when the same file contains several
500  conformations of the same molecule.
501 
502  - `--separate-chains`: Calculate SASA separately for each chain in
503  the input. Can be joined with `--separate-models` to calculate
504  SASA of each chain in each model.
505 
506  - `--chain-groups`: see @ref Chain-groups

508 @page API FreeSASA API

510 @section Basic-API Basics
511 
512 The API is found in the header [freesasa.h](freesasa_8h.html). The
513 other source-files and headers in the repository are for internal use,
514 and are not presented here, but are documented in the source
515 itself. The file [example.c](example_8c_source.html) contains a simple
516 program that illustrates how to use the API to read a PDB file from
517 `stdin` and calculate and print the SASA.
518 
519 To calculate the SASA of a structure, there are two main options:
520 
521 1. Initialize a structure from a PDB-file, using either the default
522  classifier or a custom one to determine the radius of each atom,
523  and then run the calculation.
524 
525 2. Provide an array of cartesian coordinates and an array containing
526  the radii of the corresponding atoms to freesasa\_calc\_coord().
527 
528 @subsection API-PDB Calculate SASA for a PDB file
529 
530 The following explains how to use FreeSASA to calculate the SASA of a
531 fictive PDB file (1abc.pdb). At each step one or more error checks
532 should have been done, but these are ignored here for brevity. See
533 the documentation of each function to see what errors can occur.
534 Default parameters are used at every step, the section @ref
535 Customizing explains how to configure the calculations.
536 
537 @subsubsection API-Read-PDB Open PDB file
538 
539 The function freesasa\_structure\_from\_pdb() reads the atom
540 coordinates from a PDB file and assigns a radius to each atom. The
541 third argument can be used to pass options for how to read the PDB
542 file.
543 
544 ~~~{.c}
545  FILE *fp = fopen("1abc.pdb");
546  const freesasa_classifier *classifier = &freesasa_default_classifier;
547  freesasa_structure *structure = freesasa_structure_from_pdb(fp, classifier, 0);
548 ~~~
549 
550 @subsubsection API-Calc Perform calculation and get total SASA
551 
552 Next we use freesasa\_calc\_structure() to calculate SASA using the
553 structure we just generated, and then print the total area. The argument
554 `NULL` means use default freesasa_parameters.
555 
556 ~~~{.c}
557  freesasa_result *result = freesasa_calc_structure(structure, NULL);
558  printf("Total area: %f A2\n",result->total);
559 ~~~
560 
561 @subsubsection API-Classes Get polar and apolar area
562 
563 We are commonly interested in the polar and apolar areas of a
564 molecule, this can be calculated by freesasa\_result\_classes(). To
565 get other classes of atoms we can either define our own classifier, or
566 use freesasa\_select\_area() defined in the next section. The return
567 type ::freesasa\_nodearea is a struct contains the total area and the
568 area of all apolar and polar atoms, and main-chain and side-chain
569 atoms.
570 
571 ~~~{.c}
572  freesasa_nodearea area = freesasa_result_classes(structure, result);
573  printf("Total : %f A2\n", area.total);
574  printf("Apolar : %f A2\n", area.apolar);
575  printf("Polar : %f A2\n", area.polar);
576  printf("Main-chain : %f A2\n", area.main_chain);
577  printf("Side-chain : %f A2\n", area.side_chain);
578 ~~~
579

598 @subsubsection structure-node Navigating the results as a tree
599 
600 In addition to the flat array of results in ::freesasa\_result, and
601 the global values returned by freesasa\_result\_classes(), FreeSASA
602 has an interface for navigating the results as a tree. The leaf nodes
603 are individual atoms, and there are parent nodes at the residue,
604 chain, and structure levels. The function freesasa\_calc\_tree() does
605 a SASA calculation and returns the root node of such a tree. (If one
606 already has a ::freesasa\_result the function freesasa\_tree\_init()
607 can be used instead). Each node stores a ::freesasa\_nodearea for the
608 sum of all atoms belonging to the node. The tree can be traversed with
609 freesasa\_node\_children(), freesasa\_node\_parent() and
610 freesasa\_node\_next(), and the area, type and name using
611 freesasa\_node\_area(), freesasa\_node\_type() and
612 freesasa\_node\_name(). Additionally there are special properties for
613 each level of the tree.
614 
615 @see node
616 
617 @subsubsection export-tree Exporting to RSA, JSON and XML
618 
619 The tree structure can also be exported to an RSA, JSON or XML file
620 using freesasa\_tree\_export(). The RSA format is fixed, but the user
621 can select which levels of the tree to include in JSON and XML. The
622 following illustrates how one would generate a tree and export it to
623 XML, including nodes for the whole structure, chains and residues (but
624 excluding individual atoms).
625 
626 ~~~~{.c}
627  freesasa_node *tree = freesasa_calc_tree(structure,
628  &freesasa_default_parameters,
629  &freesasa_default_classifier);
630  FILE *file = fopen("output.xml", "w");
631  freesasa_tree_export(file, tree, FREESASA_XML | FREESASA_OUTPUT_RESIDUE);
632  fclose(file);

654 @subsection Error-handling
655 
656 The principle for error handling is that unpredictable errors should
657 not cause a crash, but rather allow the user to exit gracefully or
658 make another attempt. Therefore, errors due to user or system
659 failures, such as faulty parameters, malformatted config-files, I/O
660 errors or out of memory errors, are reported through return values,
661 either ::FREESASA\_FAIL or ::FREESASA\_WARN, or by `NULL` pointers,
662 depending on the context (see the documentation for the individual
663 functions).
664 
665 Errors that are attributable to programmers using the library, such as
666 passing null pointers where not allowed, are checked by asserts.
667 
668 @subsection Thread-safety
669 
670 The only global state the library stores is the verbosity level (set
671 by freesasa\_set\_verbosity()) and the pointer to the error-log
672 (defaults to `stderr`, can be changed by freesasa\_set\_err\_out()).
673 
674 It should be clear from the documentation when the other functions
675 have side effects such as memory allocation and I/O, and thread-safety
676 should generally not be an issue (to the extent that your C library
677 has threadsafe I/O and dynamic memory allocation). The SASA
678 calculation itself can be parallelized by using a
679 ::freesasa\_parameters struct with ::freesasa\_parameters.n\_threads
680 > 1 (default is 2) where appropriate. This only gives a significant
681 effect on performance for large proteins or at high precision, and
682 because not all steps are parallelized it is usually not worth it to
683 go beyond 2 threads.

711 @subsection Classification Specifying atomic radii and classes
712 
713 Classifiers are used to determine which atoms are polar or apolar, and
714 to specify atomic radii. In addition the three standard classifiers
715 (see below) have reference values for the maximum areas of the 20
716 standard amino acids which can be used to calculate relative areas of
717 residues, as in the RSA output.
718 
719 The default classifier is available through the const variable
720 ::freesasa\_default\_classifier. This uses the *ProtOr* radii, defined
721 in the paper by Tsai et
722 al. ([JMB 1999, 290: 253](http://www.ncbi.nlm.nih.gov/pubmed/10388571))
723 for the standard amino acids (20 regular plus SEC, PYL, ASX and GLX),
724 for some capping groups (ACE/NH2) and the standard nucleic acids. If
725 the element can't be determined or is unknown, a zero radius is
726 assigned. It classes all carbons as *apolar* and all other known atoms
727 as *polar*.
728 
729 Early versions of FreeSASA used the atomic radii by Ooi et
730 al. ([PNAS 1987, 84: 3086-3090](http://www.ncbi.nlm.nih.gov/pmc/articles/PMC304812/)),
731 this classifier is still available through ::freesasa_oons_classifier.
732 
733 Users can provide their own classifiers through @ref Config-file. At
734 the moment these do not allow the user to specify reference values to
735 calculate relative SASA values for RSA output.
736 
737 The default behavior of freesasa_structure_from_pdb(),
738 freesasa_structure_array(), freesasa_structure_add_atom() and
739 freesasa_structure_add_atom_wopt() is to first try the provided
740 classifier and then guess the radius if necessary (emitting warnings
741 if this is done, uses VdW radii defined by [Mantina et al. J Phys Chem
742 2009, 113:5806](http://www.ncbi.nlm.nih.gov/pmc/articles/PMC3658832/)).
743 
744 See the documentation for these functions for what parameters to use
745 to change the default behavior.

747 @page Config-file Classifier configuration files
748 
749 The configuration files read by freesasa_classifier_from_file() or the
750 command-line option `-c` should have two sections: `types:` and
751 `atoms:`, and optionally the section `name:`.
752 
753 The types-section defines what types of atoms are available
754 (aliphatic, aromatic, hydroxyl, ...), what the radius of that type is
755 and what class a type belongs to ('polar' or 'apolar', case
756 insensitive). The types are just shorthands to associate an atom with
757 a given combination of class and radius. The user is free to define as
758 many types and classes as necessary.
759 
760 The atoms-section consists of triplets of residue-name, atom-name (as
761 in the corresponding PDB entries) and type. A prototype file would be
762 
763 ~~~
764 name: myclassifier # tag and value must be on the same line (optional)
765 
766 types:
767 C_ALIPHATIC 2.00 apolar
768 C_AROMATIC 1.75 apolar
769 N 1.55 polar
770 
771 atoms:
772 ANY N N
773 ANY CA C_ALIPHATIC
774 ANY CB C_ALIPHATIC
775 
776 ARG CG C_ALIPHATIC
777 
778 PRO CB C_AROMATIC # overrides ANY CB
779 ~~~
780 
781 The residue type `ANY` can be used for atoms that are the same in all
782 or most residues (such as backbone atoms). If there is an exception
783 for a given amino acid this can be overridden as is shown for `PRO CB`
784 in the example.
785 
786 A few example configurations are available in the directory
787 [share/](https://github.com/mittinatten/freesasa/tree/master/share). The
788 configuration-file
789 [protor.config](https://github.com/mittinatten/freesasa/tree/master/share/protor.config)
790 is a copy of the default classifier, and can be used to add extra
791 atoms that need to be classified, while keeping the defaults for the
792 standard residues (also see the file
793 [scripts/chemcomp2config.pl](https://github.com/mittinatten/freesasa/tree/master/scripts/)
794 for instructions on how to generate configurations for new chemical
795 components semi-automatically). If something common is missing in the
796 default classifier, [create an
797 issue](https://github.com/mittinatten/freesasa/issues) on Github so
798 that it can be added.
799 
800 FreeSASA also ships with some configuration-files that mimic other
801 popular programs, such as
802 [NACCESS](https://github.com/mittinatten/freesasa/tree/master/share/naccess.config)
803 and
804 [DSSP](https://github.com/mittinatten/freesasa/tree/master/share/dssp.config).

810 @page Selection Selection syntax
811 
812 FreeSASA uses a subset of the Pymol select commands to give users an
813 easy way of summing up the SASA of groups of atoms. This is done by
814 the function freesasa\_selection\_new() in the C API,
815 freesasa.selectArea() in the Python interface and the option
816 `--select` for the command line tool. All commands are case
817 insensitive. A basic selection has a selection name, a property
818 selector and a list of arguments
819 
820  <selection-name>, <selector> <list>
821 
822 For example
823 
824  aromatic, resn phe+tyr+trp+his+pro
825 
826 Several selectors can be joined using boolean logic and parentheses,
827 
828  <selection-name>, (<s1> <l1>) and not (<s2> <l2> or <s3> <l3>)
829 
830 where s1, s2 and s3 are selectors and l1, l2 and l3 are lists. The
831 operator `and` has precedence over `or`, so the second parentheses is
832 necessary but not the first, in the example above. The selection name
833 can include letters, numbers and underscores. The name can't be longer
834 than ::FREESASA\_MAX\_SELECTION\_NAME characters.
835 
836 The following property selectors are supported
837 
838 - `resn` Residue names like "ala", "arg", "du", etc
839 - `resi` Residue index (positive or negative integers)
840 - `chain` Chain labels (single characters)
841 - `name` Atom names, such as "ca", "c", "oxt", etc
842 - `symbol` Element symbols, such as "C", "O", "Se", "Fe", etc.
843 
844 A list of residues can be selected using
845 
846  resn ala+val+leu+ile+met
847 
848 and similarly for the other four selectors. In addition `resi` and
849 `chain` support ranges
850 
851  resi 1-10 (residues 1 to 10)
852  resi -10 (residues indices < 10)
853  resi 10- (residues indices > 10)
854  resi 1-10+20-30+35- (residues 1 to 10, 20 to 30 and above 35)
855  resi \-20-\-15+\-10-5 (residues -20 to -15 and -10 to 5)
856  chain A+C-E (chains A and C to E, no open intervals allowed here)
857 
858 Combining ranges with plus signs, as in the three last lines, is not
859 allowed in Pymol but supported by FreeSASA.
860 
861 If a selection list contains elements not found in the molecule that
862 is analyzed, a warning is printed and that part of the list does not
863 contribute to the selection. Not finding a list element can be because
864 it specifies a residue that does not exist in the particular molecule,
865 or because of typos. The selector does not keep a list of valid
866 elements, residue names, etc.

868 @page Geometry Geometry of Lee & Richards' algorithm
869 
870 This page explains the geometry of the calculations in L&R
871 and can be used to understand the source code. As far as possible the
872 code uses similar notation to the formulas here.
873 
874 We will use the following notation: An atom \f$i\f$ has a van der
875 Waals radius \f$r_i\f$, the rolling sphere (or *probe*) has radius
876 \f$r_\text{P}\f$ and when these are added we get an extended radius
877 \f$R_i = r_i + r_\text{P}\f$. The sphere of radius \f$R_i\f$ centered
878 at the position of atom \f$i\f$ represents the volume not accessible
879 to the center of the probe. The SASA for a molecule is then obtained
880 by calculating the non-buried surface area of the extended spheres.
881 
882 The L&R algorithm calculates the surface area by slicing the
883 protein, calculating the length of the solvent exposed contours in
884 each slice and then adding up the length multiplied by the slice
885 thickness.
886 
887 ![Slice in atom](../fig/lnr_slice.svg)
888 
889 Divide atom \f$i\f$ into \f$n\f$ slices, orthogonal to an arbitrary
890 axis, of thickness \f$\delta = 2R_i/n\f$. The position of the middle
891 of the slice along that axis is denoted \f$z\f$, and the center of
892 atom \f$i\f$, along the same axis, is at \f$z_i\f$. In each slice, the
893 atom is thus a circle of radius \f[R_i^\prime =
894 \sqrt{R_i^2-(z-z_i)^2}\,.\f] These circles are either completely
895 buried inside neighboring atoms, completely exposed, or partially
896 exposed.
897 
898 ![Overlap of circles](../fig/lnr_circles.svg)
899 
900 The exposed arc lengths for each atom can be calculated exactly. For
901 each pair of atoms \f$i,j\f$, the distance between their centers
902 projected on the slice is \f$d_{ij}\f$ (independent of \f$z\f$). If
903 \f$d_{ij} > R_i^\prime + R_j^\prime\f$, there is no overlap. If
904 \f$d_{ij} < R_j^\prime - R_i^\prime\f$ circle \f$i\f$ is completely
905 inside \f$j\f$ (and the other way around). If \f$d_{ij}\f$ lies
906 between these two cases the angle of circle \f$i\f$ that is buried due
907 to circle \f$j\f$ is
908 
909 \f[ \alpha = 2\arccos \bigl[({R_i^\prime}^2_{\,}
910 + d_{ij}^2 - {R_{j}^\prime}^2_{\,})/(2R_i^\prime d_{ij})\bigr]. \f]
911 
912 If the middle point of this arc on the circle is at an angle
913 \f$\beta\f$, the arc spans the interval
914 \f$[\beta-\alpha/2,\beta+\alpha/2]\f$. By adding up these arcs and
915 taking into account any overlap between them we get the total buried
916 angle \f$\gamma\f$ in this slices. The exposed arc angle for this atom
917 and slice is thus \f$2\pi-\gamma\f$ and the total SASA of that atom
918 
919 \f[ A_i =R_i \delta \!\! \sum_{s\in\text{slices}} \!\!
920 \left[2\pi-\gamma_s\right]\,. \f]
921 
922 The angle is multiplied by \f$R_i\f$ (not \f$R_i^\prime\f$) to give
923 the area of a conical frustum circumscribing the sphere at the
924 slice. Finally, the total area \f$A\f$ is the sum of all \f$A_i\f$.
925 
926 In FreeSASA, the L\&R SASA calculation begins by finding overlapping
927 spheres and storing the contacts in an adjacency list. It then
928 iterates through all the slices of each atom and checks for overlap
929 with adjacent atoms in each slice, and adds up the exposed arcs to
930 calculate the atom's contribution to the SASA of the slice. The
931 calculations for each atom are completely independent and can thus be
932 parallelized over an arbitrary number of threads, whereas the
933 calculation of adjacency lists has not been parallelized.
