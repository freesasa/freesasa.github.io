16 @page CLI Command-line Interface
17 
18 Building FreeSASA creates the binary `freesasa`, which is installed by
19 `make install`. Calling
20 
21  $ freesasa -h
22 
23 displays a help message listing all options. The following text
24 explains how to use most of them.
25 
26 @section CLI-Default Run using defaults
27 
28 In the following we will use the RNA/protein complex PDB structure
29 3WBM as an example. It has four protein chains A, B, C and D, and two
30 RNA strands X and Y. To run a simple SASA calculation using default
31 parameters, simply type:
32 
33  $ freesasa 3wbm.pdb
34 
35 This generates the following output
36 
37  PARAMETERS
38  algorithm : Lee & Richards
39  probe-radius : 1.400
40  threads : 2
41  slices : 20
42 
43  INPUT
44  source : 3wbm.pdb
45  chains : ABCDXY
46  atoms : 3714
47 
48  RESULTS (A^2)
49  Total : 25190.77
50  Apolar : 11552.38
51  Polar : 13638.39
52  CHAIN A : 3785.49
53  CHAIN B : 4342.33
54  CHAIN C : 3961.12
55  CHAIN D : 4904.30
56  CHAIN X : 4156.46
57  CHAIN Y : 4041.08
58 
59 The results are all in the unit Ångström-squared.
60 
61 @section parameters Changing parameters
62 
63 If higher precision is needed, the command
64 
65  $ freesasa -n 100 3wbm.pdb
66 
67 specifies that the calculation should use 100 slices per atom instead of
68 the default 20. The command
69 
70  $ freesasa --shrake-rupley -n 200 --probe-radius 1.2 --n-threads 4 3wbm.pdb
71 
72 instead calculates the SASA using Shrake & Rupley's algorithm with 200
73 test points, a probe radius of 1.2 Å, using 4 parallel threads to
74 speed things up.
75 
76 If the user wants to use their own atomic radii the command
77 
78  $ freesasa --config-file <file> 3wbm.pdb
79 
80 Reads a configuration from a file and uses it to assign atomic
81 radii. The program will halt if it encounters atoms in the PDB input
82 that are not present in the configuration. See @ref Config-file for
83 instructions how to write a configuration.
84 
85 To use the atomic radii from NACCESS call
86 
87  $ freesasa --radii=naccess 3wbm.pdb
88 
89 Another way to specify a custom set of atomic radii is to store them as
90 occupancies in the input PDB file
91 
92  $ freesasa --radius-from-occupancy 3wbm.pdb
93 
94 This option allows the user to first use either of the options
95 `--print-as-B-values` or `--B-value-file` to write generate a PDB file
96 with the radii used in the calculation, modify the radii of individual
97 atoms in that file, and the recalculate the SASA with these modified
98 radii.

127 @subsection Atom-sasa SASA for each atom
128 
129 The command-line interface can also be used as a PDB filter:
130 
131  $ cat 3wbm.pdb | freesasa --no-log --print-as-B-values
132  ATOM 1 N THR A 5 -19.727 29.259 13.573 1.64 9.44
133  ATOM 2 CA THR A 5 -19.209 28.356 14.602 1.88 5.01
134  ATOM 3 C THR A 5 -18.747 26.968 14.116 1.61 0.40
135  ...
136 
137 The output is a PDB-file where the temperature factors have been
138 replaced by SASA values (last column), and occupancy numbers by the
139 radius of each atom (second to last column).
140 
141 Only the atoms and models used in the calculation will be present in
142 the output (see @ref Input for how to modify this).

152 @subsection RSA Relative SASA values
153 
154 The CLI can also produce output similar to the RSA format from NACCESS
155 using the options `--rsa` and `--rsa-file`. This format includes both
156 absolute SASA values (ABS) and relative ones (REL) compared to a
157 precalculated reference max value. The only significant difference
158 between FreeSASA's RSA output format and that of NACCESS (except
159 differences in areas due to different atomic radii), is that FreeSASA
160 will print the value "N/A" where NACCESS prints "-99.9".
161 
162  $ freesasa 3wbm.pdb --rsa --no-log
163  ... (header)
164  REM RES _ NUM All-atoms Total-Side Main-Chain Non-polar All polar
165  REM ABS REL ABS REL ABS REL ABS REL ABS REL
166  RES THR A 5 138.48 104.1 99.58 107.4 38.90 96.3 81.59 98.1 56.89 114.0
167  RES PRO A 6 25.53 19.3 11.31 11.0 14.23 47.7 21.67 18.7 3.86 23.9
168  ...
169  RES GLY A 15 0.64 0.9 0.00 N/A 0.64 0.9 0.00 0.0 0.64 2.0
170  ...
171  RES U Y 23 165.16 N/A 165.16 N/A 0.00 N/A 52.01 N/A 113.15 N/A
172  RES C Y 24 165.01 N/A 165.01 N/A 0.00 N/A 46.24 N/A 118.77 N/A
173  RES C Y 25 262.46 N/A 262.46 N/A 0.00 N/A 85.93 N/A 176.52 N/A
174  END Absolute sums over single chains surface
175  CHAIN 1 A 3785.5 3062.1 723.3 2051.6 1733.9
176  CHAIN 2 B 4342.3 3488.6 853.7 2385.2 1957.1
177  CHAIN 3 C 3961.1 3178.5 782.7 2122.4 1838.7
178  CHAIN 4 D 4904.3 3926.8 977.5 2572.0 2332.3
179  CHAIN 5 X 4156.5 4156.5 0.0 1236.9 2919.6
180  CHAIN 6 Y 4041.1 4041.1 0.0 1184.3 2856.8
181  END Absolute sums over all chains
182  TOTAL 25190.8 21853.6 3337.2 11552.4 13638.4
183 
184 Note that each RES is a single residue, not a residue type as above
185 (i.e. has the same meaning as SEQ above). This unfortunate confusion
186 of labels is due to RSA support being added much later than the other
187 options. Fixing it now would break the interface, and will thus
188 earliest be dealt with in the next major release.
189 
190 To calculate the REL values a reference SASA value is needed,
191 calculated using the same atomic radii. At the moment such values are
192 only available for the ProtOr and NACCESS radii (selected using the
193 option `--radii`), if other radii are used all REL columns will have
194 the value 'N/A'.
195 
196 The reference SASA values for residue X are calculated from Ala-X-Ala
197 peptides in a stretched out configuration. The reference
198 configurations are supplied for reference in the directory
199 `rsa`. Since these are not always the most exposed possible
200 configuration, and because bond lengths and bond angles vary, the
201 relative SASA values will sometimes be larger than 100 %. At the
202 moment there is no interface to supply user-defined reference values.
203 
204 @subsubsection RSA-naccess Using the NACCESS configuration
205 
206 The reference values for the NACCESS configuration in FreeSASA are not
207 exactly the same as those that ship with NACCESS, but have been
208 calculated from scratch using the tripeptides that ship with
209 FreeSASA. Calling
210 
211  $ freesasa 3wbm.pdb --rsa-file=3wbm.rsa --radii=naccess --no-log
212 
213 will give an RSA file where the ABS columns are identical to NACCESS
214 (run with the flag `-b`) for the amino acids, and for nucleic acids
215 the definition of backbone differs (no nucleic acid backbone has been
216 defined in FreeSASA). REL values will differ slightly, due to the
217 differences in reference values above.

219 @section CLI-select Selecting groups of atoms
220 
221 The option `--select` can be used to define groups of atoms whose
222 integrated SASA we are interested in. It uses a subset of the Pymol
223 `select` command syntax, see @ref Selection for full
224 documentation. The following example shows how to calculate the sum of
225 exposed surface areas of all aromatic residues and of the four chains
226 A, B, C and D (just the sum of the areas above).
227 
228  $ freesasa --select "aromatic, resn phe+tyr+trp+his+pro" --select "abcd, chain A+B+C+D" 3wbm.pdb
229  ...
230  SELECTIONS
231  freesasa: warning: Found no matches to resn 'TRP', typo?
232  freesasa: warning: Found no matches to resn 'HIS', typo?
233  aromatic : 1196.45
234  abcd : 16993.24
235 
236 The lines shown above are appended to the regular output. This
237 particular protein did not have any TRP or HIS residues, hence the
238 warnings (written to stderr). The warnings can be supressed with the
239 flag `-w`.

311 @section Input PDB input

313 @subsection Hetatom-hydrogen Including extra atoms
314 
315 The user can ask to include hydrogen atoms and HETATM entries in the
316 calculation using the options `--hydrogen` and `--hetatm`. In both
317 cases adding unknown atoms will emit a warning for each atom. This can
318 either be amended by using the flag '-w' to suppress warnings, or by
319 using a custom classifier so that they are recognized (see @ref
320 Config-file).
321 
322 @subsection Halt-skip Skipping unknown atoms
323 
324 By default FreeSASA guesses the element of an unknown atom and uses
325 that elements VdW radius. If this fails the radius is set to 0 (and
326 hence the atom will not contribute to the calculated area). Users can
327 request to either skip unknown atoms completely (i.e. no guessing) or
328 to halt when unknown atoms are found and exit with an error. This is
329 done with the option `--unknown` which takes one of the three
330 arguments `skip`, `halt` or `guess` (default). Whenever an unknown
331 atom is skipped or its radius is guessed a warning is printed to
332 stderr.
333 
334 @subsection Chains-models Separating and joining chains and models
335 
336 If a PDB file has several chains and/or models, by default all chains
337 of the first model are used, and the rest of the file is ignored. This
338 behavior can be modified using the following options
339 
340  - `--join-models`: Joins all models in the input into one large
341  structure. Useful for biological assembly files were different
342  locations of the same chain in the oligomer are represented by
343  different MODEL entries.
344 
345  - `--separate-models`: Calculate SASA separately for each model in
346  the input. Useful when the same file contains several
347  conformations of the same molecule.
348 
349  - `--separate-chains`: Calculate SASA separately for each chain in
350  the input. Can be joined with `--separate-models` to calculate
351  SASA of each chain in each model.
352 
353  - `--chain-groups`: see @ref Chain-groups

355 @page API FreeSASA API
356 
357 @section Basic-API Basics
358 
359 The API is found in the header [freesasa.h](freesasa_8h.html) and this
360 is the only header installed by `make install`. The other source-files
361 and headers in the repository are for internal use, and are not
362 present here, but are documented in the source itself. The file
363 [example.c](example_8c_source.html) contains a simple program that
364 illustrates how to use the API to read a PDB file from `stdin` and
365 calculate and print the SASA.
366 
367 To calculate the SASA of a structure, there are two main options:
368 
369 1. Initialize a structure from a PDB-file, using either the default
370  classifier or a custom one to determine the radius of each atom,
371  and then run the calculation.
372 
373 2. Provide an array of cartesian coordinates and an array containing
374  the radii of the corresponding atoms to freesasa_calc_coord().
375 
376 @subsection API-PDB Calculate SASA for a PDB file
377 
378 The following explains how to use FreeSASA to calculate the SASA of a
379 fictive PDB file (1abc.pdb). At each step one or more error checks
380 should have been done, but these are ignored here for brevity. See
381 the documentation of each function to see what errors can occur.
382 Default parameters are used at every step, the section @ref
383 Customizing explains how to configure the calculations.
384 
385 @subsubsection API-Read-PDB Open PDB file
386 
387 The function freesasa_structure_from_pdb() reads the atom coordinates
388 from a PDB file and assigns a radius to each atom. The second and
389 third arguments can be changed to use a custom freesasa_classifier to
390 define radii and to specify options for what to include in the PDB
391 file, respectively.
392 
393 ~~~{.c}
394  FILE *fp = fopen("1abc.pdb");
395  freesasa_structure *structure = freesasa_structure_from_pdb(fp, NULL, 0);
396 ~~~
397 
398 @subsubsection API-Calc Perform calculation and get total SASA
399 
400 Next we use freesasa_calc_structure() to calculate SASA using the
401 structure we just generated, and then print the total area. The argument
402 `NULL` means use default freesasa_parameters.
403 
404 ~~~{.c}
405  freesasa_result *result = freesasa_calc_structure(structure, NULL);
406  printf("Total area: %f A2\n",result->total);
407 ~~~
408 
409 @subsubsection API-Classes Get polar and apolar area
410 
411 We are commonly interested in the polar and apolar areas of a
412 molecule, this can be calculated by freesasa_result_classify(). Again,
413 passing a NULL freesasa_classifier uses the default. To get other
414 classes of atoms we can either define our own classifier, or use
415 freesasa_select_area() defined in the next section. The return type
416 freesasa_strvp contains arrays of strings and values describing the
417 different classes, as illustrated below.
418 
419 ~~~{.c}
420  freesasa_strvp *class_area = freesasa_result_classify(result, structure, NULL);
421  for (int i = 0; i < class_area->n; ++i)
422  printf("%s: %f A2\n", class_area->string[i], class_area->value[i]);
423 ~~~
424 
425 @see @ref Classification
426 
427 @subsubsection API-Select Get area of custom groups of atoms
428 
429 Groups of atoms can be defined using freesasa_select_area(), which
430 takes a selection definition uses a subset of the Pymol select syntax
431 
432 ~~~{.c}
433  double area;
434  char name[FREESASA_MAX_SELECTION_NAME+1];
435  freesasa_select_area("aromatic, resn phe+tyr+trp+his+pro",
436  name, &area, structure, result);
437  printf("Area of selection '%s': %f A2\n", name, area);
438 ~~~
439 
440 @see @ref Selection.
441 
442 @subsection Coordinates
443 
444 If users wish to supply their own coordinates and radii, these are
445 accepted as arrays of doubles passed to the function
446 freesasa_calc_coord(). The coordinate-array should have size 3*n with
447 coordinates in the order `x1,y1,z1,x2,y2,z2,...,xn,yn,zn`.
448 
449 ~~~{.c}
450  double coord[] = {/* x */ 1.0, /* y */ 2.0, /* z */ 3.0};
451  double radius[] = {2.0};
452  freesasa_result *result = freesasa_calc_coord(coord, radius, 1, NULL);
453 ~~~
454 
455 @subsection Error-handling
456 
457 The principle for error handling is that unpredictable errors should
458 not cause a crash, but rather allow the user to exit gracefully or
459 make another attempt. Therefore, errors due to user or system
460 failures, such as faulty parameters, malformatted config-files, I/O
461 errors or out of memory errors, are reported through return values,
462 either ::FREESASA_FAIL or ::FREESASA_WARN, or by NULL pointers,
463 depending on the context. See the documentation for the individual
464 functions. If memory allocation fails as much memory as possible is
465 released. To the extent that it's possible to emulate system failures
466 like this, they have been verified to not cause aborts or seg-faults
467 (see the tests/ directory) in any part of the code.
468 
469 Errors that are attributable to programmers using the library, such as
470 passing null pointers where not allowed, are checked by asserts.
471 
472 @subsection Thread-safety
473 
474 The only global state the library stores is the verbosity level (set
475 by freesasa_set_verbosity()) and the pointer to the error-log
476 (defaults to `stderr`, can be changed by freesasa_set_err_out()).
477 
478 It should be clear from the documentation when the other
479 functions have side effects such as memory allocation and I/O, and
480 thread-safety should generally not be an issue (to the extent that
481 your C library has threadsafe I/O and dynamic memory allocation). The
482 SASA calculation itself can be parallelized by passing a
483 ::freesasa_parameters struct with ::freesasa_parameters.n_threads set
484 to a value > 1 (default is 2) to freesasa_calc_structure() or
485 freesasa_calc_coord(). This only gives a significant effect on
486 performance for large proteins or at high precision, and because not
487 all steps are parallelized it is usually not worth it to go beyond 2
488 threads.

490 @section Customizing Customizing behavior
491 
492 The types ::freesasa_parameters and ::freesasa_classifier can be used
493 to change the parameters of the calculations. Users who wish to use
494 the defaults can pass NULL wherever pointers to these are requested.
495 
496 @subsection Parameters Parameters
497 
498 Changing parameters is done by passing a ::freesasa_parameters object
499 with the desired values. It can be initialized to default by
500 
501 ~~~{.c}
502 freesasa_parameters param = freesasa_default_parameters;
503 ~~~
504 
505 To allow the user to only change the parameters that are non-default.
506 
507 The following call would run a high precision Shrake & Rupley
508 calculation with 10000 test points
509 
510 ~~~{.c}
511 param.alg = FREESASA_SHRAKE_RUPLEY;
512 param.shrake_rupley_n_points = 10000;
513 freesasa_result *result = freesasa_calc_structure(structure,radii,param);
514 ~~~
515 
516 @subsection Classification Specifying atomic radii and classes
517 
518 The type ::freesasa_classifier has function pointers to functions that
519 take residue and atom names as argument (pairs such as "ALA","CA"),
520 and returns a radius or a class (polar, apolar, etc). The classifier
521 can be passed to freesasa_structure_from_pdb(),
522 freesasa_structure_array() or freesasa_structure_add_atom_wopt() to
523 customize the radii assigned to atoms, which can then be used to
524 calculate the SASA of the structure. It can also be used in
525 freesasa_result_classify() to get the SASA integrated over the
526 different classes of atoms, i.e. the SASA of all polar atoms, etc.
527 
528 Users of the API can provide their own classification by writing their
529 own functions and providing them via a ::freesasa_classifier object. A
530 classifier-configuration can also be read from a file using
531 freesasa_classifier_from_file() (see @ref Config-file).
532 
533 The default classifier is available through the const variable
534 ::freesasa_default_classifier. This uses the radii, defined in the
535 paper by Tsai et al. ([JMB 1999, 290:
536 253](http://www.ncbi.nlm.nih.gov/pubmed/10388571)) for the standard
537 amino acids (20 regular plus SEC, PYL, ASX and GLX), for some capping
538 groups (ACE/NH2) and the nucleic acids. If the element can't be
539 determined or is unknown, a negative radius is returned. It classes
540 all carbons as *apolar* and all other known atoms as *polar*.
541 
542 Early versions of FreeSASA used the atomic radii by Ooi et al. ([PNAS
543 1987, 84: -3086](http://www.ncbi.nlm.nih.gov/pmc/articles/PMC304812/),
544 this classifier is still available through freesasa_classifier_oons().
545 
546 The default behavior of freesasa_structure_from_pdb(),
547 freesasa_structure_array(), freesasa_structure_add_atom() and
548 freesasa_structure_add_atom_wopt() is to first try the default
549 classifier and then guess the radius if necessary (emitting warnings
550 if this is done, uses VdW radii defined by [Mantina et al. J Phys Chem
551 2009,
552 113:5806](http://www.ncbi.nlm.nih.gov/pmc/articles/PMC3658832/)).
553 
554 See the documentation for these functions for what
555 parameters to use to change the default behavior.

557 @page Config-file Classifier configuration files
558 
559 The configuration files read by freesasa_classifier_from_file() or the
560 command-line option `-c` should have two sections: `types:` and
561 `atoms:`.
562 
563 The types-section defines what types of atoms are available
564 (aliphatic, aromatic, hydroxyl, ...), what the radius of that type is
565 and what class a type belongs to (polar, apolar, ...). The types are
566 just shorthands to associate an atom with a given combination of class
567 and radius. The user is free to define as many types and classes as
568 necessary.
569 
570 The atoms-section consists of triplets of residue-name, atom-name (as
571 in the corresponding PDB entries) and type. A prototype file would be
572 
573 ~~~
574 types:
575 C_ALIPHATIC 2.00 apolar
576 C_AROMATIC 1.75 apolar
577 N 1.55 polar
578 
579 atoms:
580 ANY N N
581 ANY CA C_ALIPHATIC
582 ANY CB C_ALIPHATIC
583 
584 ARG CG C_ALIPHATIC
585 
586 PRO CB C_AROMATIC # overrides ANY CB
587 ~~~
588 
589 The residue type `ANY` can be used for atoms that are the same in all
590 or most residues (such as backbone atoms). If there is an exception
591 for a given amino acid this can be overridden as is shown for `PRO CB`
592 in the example.
593 
594 A few example configurations are available in the directory
595 [share/](https://github.com/mittinatten/freesasa/tree/master/share). The
596 configuration-file
597 [protor.config](https://github.com/mittinatten/freesasa/tree/master/share/protor.config)
598 is a copy of the default classifier, and can be used to add extra
599 atoms that need to be classified, while keeping the defaults for the
600 standard residues (also see the file
601 [scripts/chemcomp2config.pl](https://github.com/mittinatten/freesasa/tree/master/scripts/)
602 for instructions on how to generate configurations for new chemical
603 components semi-automatically). If something common is missing in the
604 default classifier, [create an
605 issue](https://github.com/mittinatten/freesasa/issues) on Github so
606 that it can be added.
607 
608 FreeSASA also ships with some configuration-files that mimic other
609 popular programs, such as
610 [NACCESS](https://github.com/mittinatten/freesasa/tree/master/share/naccess.config)
611 and

614 @page Selection Selection syntax
615 
616 FreeSASA uses a subset of the Pymol select commands to give users an
617 easy way of summing up the SASA of groups of atoms. This is done by
618 the function freesasa_select_area() in the C API,
619 freesasa.selectArea() in the Python interface and the option
620 `--select` for the command line tool. All commands are case
621 insensitive. A basic selection has a selection name, a property
622 selector and a list of arguments
623 
624  <selection-name>, <selector> <list>
625 
626 For example
627 
628  aromatic, resn phe+tyr+trp+his+pro
629 
630 Several selectors can be joined using boolean logic and parentheses,
631 
632  <selection-name>, (<s1> <l1>) and not (<s2> <l2> or <s3> <l3>)
633 
634 where s1, s2 and s3 are selectors and l1, l2 and l3 are lists. The
635 operator `and` has precedence over `or`, so the second parentheses is
636 necessary but not the first, in the example above. The selection name
637 has to start with a letter, but it can include numbers and
638 underscores. The name can't be longer than
639 ::FREESASA_MAX_SELECTION_NAME characters.
640 
641 The following property selectors are supported
642 
643 - `resn` Residue names like "ala", "arg", "du", etc
644 - `resi` Residue index (positive integers)
645 - `chain` Chain labels (single characters)
646 - `name` Atom names, such as "ca", "c", "oxt", etc
647 - `symbol` Element symbols, such as "C", "O", "Se", "Fe", etc.
648 
649 A list of residues can be selected using
650 
651  resn ala+val+leu+ile+met
652 
653 and similarly for the other four selectors. In addition `resi` and
654 `chain` support ranges
655 
656  resi 1-10
657  resi 1-10+20-30+35
658  chain A+C-E
659 
660 Combining ranges with plus signs, as in the two last lines, is not
661 allowed in Pymol but supported by FreeSASA.
662 
663 If a selection list contains elements not found in the molecule that
664 is analyzed, a warning is printed and that part of the list does not
665 contribute to the selection. Not finding an a list element can be
666 because it specifies a residue that does not exist in the particular
667 molecule, or because of typos. The selector does not keep a list of
668 valid elements, residue names, etc.

727 @section Python-classification Customizing atom classification
728 
729 This uses the NACCESS parameters (the file 'naccess.config' is
730 available in the share/ directory of the repository).
731 
732 ~~~{.py}
733 classifier = freesasa.Classifier("naccess.config")
734 structure = freesasa.Structure("1ubq.pdb",classifier)
735 result = freesasa.calc(structure)
736 area_classes = freesasa.classifyResults(result,structure,classifier)
737 ~~~
738 
739 Classification can be customized also by extending the Classifier
740 interface. The code below is an illustration of a classifier that
741 classes Nitrogens separately, and assigns radii based on element only
742 (and crudely).
743 
744 ~~~{.py}
745 import freesasa
746 import re
747 
748 class DerivedClassifier(Classifier):
749  def classify(self,residueName,atomName):
750  if re.match('\s*N',atomName):
751  return 'Nitrogen'
752  return 'Not-nitrogen'
753 
754  def radius(self,residueName,atomName):
755  if re.match('\s*N',atomName): # Nitrogen
756  return 1.6
757  if re.match('\s*C',atomName): # Carbon
758  return 1.7
759  if re.match('\s*O',atomName): # Oxygen
760  return 1.4
761  if re.match('\s*S',atomName): # Sulfur
762  return 1.8
763  return 0; # everything else (Hydrogen, etc)
764 
765 classifier = DerivedClassifier()
766 
767 # use the DerivedClassifier to calculate atom radii
768 structure = freesasa.Structure("1ubq.pdb",classifier)
769 result = freesasa.calc(structure)
770 
771 # use the DerivedClassifier to classify atoms
772 area_classes = freesasa.classifyResults(result,structure,classifier)
773 ~~~
774 
775 Of course, this example is somewhat contrived, if we only want the
776 integrated area of Nitrogen atoms, the simpler choice would be
777 ~~~{.py}
778  selection = freesasa.selectArea('nitrogen, symbol n', structure, result)
779 ~~~
780 
781 However, extending freesasa.Classifier, as illustrated above, allows
782 classification to arbitrary complexity and also lets us redefine the
783 radii used in the calculation.

801 @page Geometry Geometry of Lee & Richards' algorithm
802 
803 This page explains the geometry of the calculations in L&R
804 and can be used to understand the source code. As far as possible the
805 code uses similar notation to the formulas here.
806 
807 We will use the following notation: An atom \f$i\f$ has a van der
808 Waals radius \f$r_i\f$, the rolling sphere (or *probe*) has radius
809 \f$r_\text{P}\f$ and when these are added we get an extended radius
810 \f$R_i = r_i + r_\text{P}\f$. The sphere of radius \f$R_i\f$ centered
811 at the position of atom \f$i\f$ represents the volume not accessible
812 to the center of the probe. The SASA for a molecule is then obtained
813 by calculating the non-buried surface area of the extended spheres.
814 
815 The L&R algorithm calculates the surface area by slicing the
816 protein, calculating the length of the solvent exposed contours in
817 each slice and then adding up the length multiplied by the slice
818 thickness.
819 
820 ![Slice in atom](../fig/lnr_slice.svg)
821 
822 Divide atom \f$i\f$ into \f$n\f$ slices, orthogonal to an arbitrary
823 axis, of thickness \f$\delta = 2R_i/n\f$. The position of the middle
824 of the slice along that axis is denoted \f$z\f$, and the center of
825 atom \f$i\f$, along the same axis, is at \f$z_i\f$. In each slice, the
826 atom is thus a circle of radius \f[R_i^\prime =
827 \sqrt{R_i^2-(z-z_i)^2}\,.\f] These circles are either completely
828 buried inside neighboring atoms, completely exposed, or partially
829 exposed.
830 
831 ![Overlap of circles](../fig/lnr_circles.svg)
832 
833 The exposed arc lengths for each atom can be calculated exactly. For
834 each pair of atoms \f$i,j\f$, the distance between their centers
835 projected on the slice is \f$d_{ij}\f$ (independent of \f$z\f$). If
836 \f$d_{ij} > R_i^\prime + R_j^\prime\f$, there is no overlap. If
837 \f$d_{ij} < R_j^\prime - R_i^\prime\f$ circle \f$i\f$ is completely
838 inside \f$j\f$ (and the other way around). If \f$d_{ij}\f$ lies
839 between these two cases the angle of circle \f$i\f$ that is buried due
840 to circle \f$j\f$ is
841 
842 \f[ \alpha = 2\arccos \bigl[({R_i^\prime}^2_{\,}
843 + d_{ij}^2 - {R_{j}^\prime}^2_{\,})/(2R_i^\prime d_{ij})\bigr]. \f]
844 
845 If the middle point of this arc on the circle is at an angle
846 \f$\beta\f$, the arc spans the interval
847 \f$[\beta-\alpha/2,\beta+\alpha/2]\f$. By adding up these arcs and
848 taking into account any overlap between them we get the total buried
849 angle \f$\gamma\f$ in this slices. The exposed arc angle for this atom
850 and slice is thus \f$2\pi-\gamma\f$ and the total SASA of that atom
851 
852 \f[ A_i =R_i \delta \!\! \sum_{s\in\text{slices}} \!\!
853 \left[2\pi-\gamma_s\right]\,. \f]
854 
855 The angle is multiplied by \f$R_i\f$ (not \f$R_i^\prime\f$) to give
856 the area of a conical frustum circumscribing the sphere at the
857 slice. Finally, the total area \f$A\f$ is the sum of all \f$A_i\f$.
858 
859 In FreeSASA, the L\&R SASA calculation begins by finding overlapping
860 spheres and storing the contacts in an adjacency list. It then
861 iterates through all the slices of each atom and checks for overlap
862 with adjacent atoms in each slice, and adds up the exposed arcs to
863 calculate the atom's contribution to the SASA of the slice. The
864 calculations for each atom are completely independent and can thus be
865 parallelized over an arbitrary number of threads, whereas the
866 calculation of adjacency lists has not been parallelized.
